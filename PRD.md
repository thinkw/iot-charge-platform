## 一、项目概述
### 1.1 项目背景
随着新能源汽车保有量快速增长，智能充电设施运营需求激增。传统充电管理系统普遍存在设备接入能力弱、状态同步不及时、计费规则僵化、异常处理能力差等问题。

本项目从零构建一套轻量级IoT充电运营平台，采用行业标准的MQTT+Netty技术栈实现设备通信，结合成熟的高并发交易系统架构，解决海量设备长连接管理、设备与订单状态一致性、分时计费、远程管控等核心痛点。项目通过软件模拟器替代实体硬件，纯软件验证全流程能力，为后续扩展微服务架构和真实硬件接入预留接口。

### 1.2 项目目标
1. **业务目标**：实现设备接入、用户充电、订单计费、实时监控、故障告警全业务闭环，支持预约充电、扫码启桩、远程启停、峰谷计费等核心功能。
2. **技术目标**：单机支撑≥100台虚拟充电桩并发长连接，核心接口响应时间<200ms，并发场景无超卖、无状态错乱、无数据丢失。
3. **演示目标**：配套独立虚拟充电桩模拟器，完整模拟设备上下线、数据上报、指令响应、故障触发，全流程可现场演示。
4. **学习目标**：掌握IoT云平台核心技术（MQTT/Netty、状态机、长连接治理），验证Java21虚拟线程在高并发场景的性能优势。

### 1.3 项目范围
#### 包含范围
- 设备接入与管理：MQTT协议接入、设备鉴权、心跳保活、状态上报、远程控制
- 用户端服务：注册登录、站点查询、充电桩预约、扫码启桩、实时充电查看、订单管理、支付结算
- 运营后台：充电站管理、充电桩档案、订单统计、计费规则配置、设备告警查看、数据大屏
- 基础能力：Redis分布式锁、RocketMQ异步解耦、WebSocket实时推送、定时任务、幂等处理
- 辅助工具：独立虚拟充电桩模拟器，支持批量模拟设备行为

#### 排除范围
- 实体充电桩硬件开发、硬件驱动、底层电路设计
- 大规模集群部署、分布式事务、异地多活
- 充电桩固件升级、边缘计算、车机互联
- 复杂营销活动（满减、优惠券、积分）、多商户入驻

## 三、核心业务流程
### 3.1 设备接入流程
1. 虚拟充电桩启动，向平台发送注册请求（携带设备唯一标识SN）
2. 平台验证设备合法性，返回接入凭证和MQTT连接参数
3. 设备建立MQTT长连接，发送心跳报文（30秒/次）
4. 平台维护设备在线状态，超时未收到心跳标记为离线
5. 设备定时上报状态（空闲/充电/故障）和实时数据（电压、电流、功率、电量）

### 3.2 扫码充电主流程
1. 用户扫描充电桩二维码，小程序跳转至充电确认页
2. 平台验证充电桩状态（空闲/在线）
3. 用户确认充电，平台通过Redis分布式锁锁定该充电桩
4. 平台向设备下发"启动充电"指令
5. 设备接收指令，开始充电，上报充电状态和实时数据
6. 平台通过WebSocket向用户端推送充电进度和实时费用
7. 用户点击"结束充电"或充电完成，平台向设备下发"停止充电"指令
8. 设备停止充电，上报最终充电数据
9. 平台计算费用，生成订单，发起支付
10. 用户支付完成，订单状态更新为已完成

### 3.3 预约充电流程
1. 用户选择充电站、充电桩和预约时段
2. 平台验证该时段充电桩是否可用
3. 平台通过Redis分布式锁锁定该时段的充电桩
4. 用户支付预约押金，生成预约订单
5. 预约时间到达前15分钟，平台向用户发送提醒
6. 用户在预约时间内扫码启桩，自动抵扣押金
7. 超时未使用，自动取消预约，退还押金（扣除违约金）

### 3.4 故障告警流程
1. 设备检测到故障（过温、过压、短路、离线），上报故障信息
2. 平台接收故障报文，通过RocketMQ异步处理
3. 平台更新设备状态为故障，生成告警记录
4. 平台通过WebSocket向运营后台推送告警信息
5. 运营人员处理告警，标记为已处理
6. 设备故障排除后，上报恢复信息，平台更新设备状态为空闲

## 四、详细功能需求
### 4.1 设备接入与管理模块（核心IoT模块）
#### 4.1.1 设备注册与鉴权
- 支持设备通过SN+密钥方式注册
- 生成设备唯一接入凭证
- 验证设备合法性，拒绝非法设备接入
- 记录设备注册时间、最后上线时间

#### 4.1.2 长连接管理
- 基于Netty+MQTT实现设备长连接
- 心跳检测：30秒心跳间隔，90秒未收到心跳标记为离线
- 断线重连：设备断线后自动重连，恢复状态
- 连接隔离：设备连接与业务接口资源隔离，避免相互影响
- 连接统计：实时统计在线设备数、总连接数

#### 4.1.3 状态上报与管理
- 设备上报状态：空闲、充电中、故障、离线、锁定
- 设备上报实时数据：电压(V)、电流(A)、功率(kW)、已充电量(kWh)、温度(℃)
- 平台维护设备全局状态，存储于Redis
- 状态变更记录：记录所有设备状态变更历史
- 状态机管控：严格限制状态流转，禁止非法状态变更

#### 4.1.4 远程控制
- 支持远程下发启动充电指令
- 支持远程下发停止充电指令
- 支持远程重启设备
- 支持远程设置设备参数（如最大功率）
- 指令下发确认：设备接收指令后返回确认报文，超时未确认重试3次

#### 4.1.5 故障告警
- 支持设备主动上报故障
- 支持平台检测故障（如离线超时）
- 故障类型：过温、过压、欠压、短路、漏电、离线、通信异常
- 告警级别：一般、重要、紧急
- 告警通知：运营后台实时弹窗、声音提醒

### 4.2 用户端模块
#### 4.2.1 账号管理

## 五、非功能需求
### 5.1 性能需求
- 单机支持≥100台虚拟充电桩并发长连接
- 核心接口（启桩、停桩、下单）响应时间<200ms
- 普通查询接口响应时间<100ms
- 设备状态上报处理延迟<1秒
- WebSocket实时推送延迟<500ms
- 支持≥50并发用户同时操作

### 5.2 可用性需求
- 系统可用性≥99.9%
- 单节点故障不影响整体系统运行（单体架构下尽量保证）
- 数据备份：每日自动备份MySQL数据库
- 故障恢复：系统崩溃后能在5分钟内重启恢复

### 5.3 安全性需求
- 所有接口需进行身份认证
- 设备接入需进行鉴权，防止非法设备接入
- 敏感数据（用户密码、支付信息）加密存储
- 防止SQL注入、XSS攻击、CSRF攻击
- 接口限流：防止恶意请求攻击

### 5.4 可扩展性需求
- 模块化设计，模块间低耦合
- 预留微服务拆分接口，后续可平滑迁移
- 计费规则插件化设计，方便新增计费方式
- 支持新增设备类型和协议

### 5.5 兼容性需求
- 支持Chrome、Firefox、Edge等主流浏览器
- 支持微信小程序、H5页面
- 支持MQTT 3.1.1协议
- 支持MySQL 8.0及以上版本
- 支持Redis 7.0及以上版本

## 六、技术架构设计
### 6.1 整体架构
采用分层单体架构，分为接入层、业务层、数据层和设备仿真层，各层通过接口交互，低耦合高内聚。

```
┌─────────────────────────────────────────────────────────┐
│                    设备仿真层                            │
│  虚拟充电桩模拟器 → MQTT客户端 → 模拟硬件行为            │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                    接入层                                │
│  Netty+MQTT(设备长连接)  HTTP(用户/后台接口)  WebSocket(实时推送) │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                    业务层                                │
│  设备服务  用户服务  订单服务  计费服务  告警服务  运营服务  │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│                    数据层                                │
│  MySQL(业务数据)  Redis(状态/缓存/锁)  RocketMQ(事件队列) │
└─────────────────────────────────────────────────────────┘
```

### 6.3 中间件使用场景
- **Redis**：
  - 存储设备在线状态、实时数据
  - 实现分布式锁（抢桩、预约时段锁）
  - 缓存热点数据（站点信息、计费规则）
  - 接口限流
- **RocketMQ**：
  - 设备事件队列：设备上下线、状态变更、故障上报
  - 订单事件队列：订单创建、支付完成、充电结束
  - 延时队列：预约超时取消、充电时长超限断电
  - 告警事件队列：故障告警通知
- **Netty+MQTT**：
  - 设备长连接管理
  - MQTT协议报文编解码
  - 设备消息收发
- **WebSocket**：
  - 运营大屏实时数据推送
  - 用户端充电进度推送
  - 告警信息实时推送

## 十一、风险评估与应对
| 风险                   | 可能性 | 影响程度 | 应对措施                                                     |
| ---------------------- | ------ | -------- | ------------------------------------------------------------ |
| MQTT/Netty通信调试困难 | 中     | 高       | 使用MQTT.fx工具辅助调试，分步打通单设备通信，再扩展批量设备  |
| 设备与订单状态不一致   | 中     | 高       | 严格使用状态机管控状态，异常事件通过MQ异步补偿，定期巡检数据 |
| 并发抢桩超卖           | 低     | 高       | 使用Redis分布式锁实现设备+时段双重锁定，数据库乐观锁兜底     |
| 长连接资源耗尽         | 低     | 中       | 实现空闲连接自动销毁，连接数限制，资源隔离                   |
| 项目范围蔓延           | 中     | 中       | 严格按照PRD范围开发，非核心功能后续迭代                      |

## 十二、验收标准
1. **功能验收**：所有PRD中规划的功能正常运行，虚拟模拟器与平台交互正常，全业务流程可演示。
2. **性能验收**：单机支持≥100台虚拟设备长连接，核心接口响应时间<200ms，并发场景无超卖、无状态错乱。
3. **质量验收**：代码规范，无严重bug，单元测试覆盖率≥60%。
4. **文档验收**：源码、数据库脚本、部署文档、接口文档、项目说明文档齐全。
